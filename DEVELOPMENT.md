# Development

## Quick start

```bash
npm install
npm run demo            # generates apps/web/public/demo-run.json (mock adapter)
npm run dev             # http://127.0.0.1:5175/
```

The dashboard reads `apps/web/public/demo-run.json` (regenerated by
`npm run demo`) plus the per-agent `apps/web/public/agents/*.json` files
(also generated by the runner).

## Workspaces

```
apps/agents       deterministic multi-agent runner            (@clawdao/agents)
apps/web          live dashboard (Vite + React 19)            (@clawdao/web)
packages/core     types, ABIs, wallets, executors             (@clawdao/core)
scripts           CLI entrypoints for chain interactions      (@clawdao/scripts)
contracts         Foundry project for AgenticTreasury         (no npm package)
```

## Commands

| Command | What it does |
|---|---|
| `npm run demo` | Runs the agent demo runner, writes `demo-run.json` and 5 agent registration files. |
| `npm run dev` | Vite dev server on port 5175. |
| `npm run build` | TypeScript noEmit + Vite production build. |
| `npm run typecheck` | tsc --noEmit across all 4 workspaces. |
| `npm run check` | typecheck + build. |
| `DRY_RUN=1 npm run register` | Prints register() calldata + the data: URI for each agent. No tx broadcast. |
| `npm run register` | Broadcasts 5 `register()` txs to Mantle Sepolia ERC-8004 IdentityRegistry; writes `apps/web/public/agent-ids.json`. |
| `DRY_RUN=1 npm run feedback` | Prints `giveFeedback()` calldata. |
| `npm run feedback` | Broadcasts 5 `giveFeedback()` txs; avoids ERC-8004 self-feedback. |
| `DRY_RUN=1 npm run validate` | Prints `validationRequest` + `validationResponse` calldata. |
| `npm run validate` | Broadcasts the validation request (Claw / subject owner → Sentinel) and response (Sentinel) for the most recent cycle. |
| `npm run verify` | Reads back tokenURI / ownerOf / getSummary for each registered agent. |
| `npm run poll-chain` | Continuously snapshots Mantle Sepolia state into `apps/web/public/live-chain.json` (every 30s by default). |
| `POLL_ONCE=1 npm run poll-chain` | One snapshot then exit. |
| `GENERATE_MNEMONIC=1 npm run wallets` | Generate `.env.generated` with a fresh five-agent mnemonic and print derived public addresses. |
| `npm run wallets` | Print the five configured agent public addresses from env. |
| `npm run preflight` | Read-only check for local demo artifacts, logo, env, agent IDs, and chain snapshot. |
| `npm run compile-treasury` | Compile AgenticTreasury with npm `solc`; writes `contracts/out/.../AgenticTreasury.json`. |
| `DRY_RUN=1 npm run deploy-treasury` | Print constructor args + bytecode size; no chain call. |
| `npm run deploy-treasury` | Deploy AgenticTreasury from the compiled artifact, write `apps/web/public/deployed-treasury.json`, append `TREASURY_ADDRESS=` to `.env`. |
| `npm run watch-events` | Backfill last `WATCH_LOOKBACK_BLOCKS` then subscribe to Identity / Reputation / Validation / Treasury events; writes `apps/web/public/events.json`. |
| `WATCH_ONCE=1 npm run watch-events` | Backfill once and exit. |
| `WATCH_FIXTURE=1 npm run watch-events` | Synthetic events.json (no RPC) for offline dashboard preview. |
| `npm run live-demo` | Wraps demo + (optional) deploy + register + feedback + validate + watch. |
| `npm run live-demo -- --deploy --register --feedback --validate --watch` | Full on-chain rehearsal in one command. |
| `FORCE_SCENARIO=oversized-action npm run demo` | Force a specific scenario for the next cycle. |

## Environment variables

Copy `.env.example` to `.env` and fill in.

| Var | Used by | Purpose |
|---|---|---|
| `AGENT_MNEMONIC` | wallets.ts | Derive 5 agent accounts at `m/44'/60'/0'/0/{0..4}`. |
| `SCOUT/GUARD/CLAW/LEDGER/SENTINEL_PRIVATE_KEY` | wallets.ts | Per-agent override (highest priority). |
| `MANTLE_SEPOLIA_RPC_URL` | scripts | Mantle Sepolia JSON-RPC endpoint. |
| `TREASURY_ADDRESS` | mantle-sepolia adapter | Deployed AgenticTreasury contract. |
| `EXECUTOR` | demo-runner | `auto` / `mock` / `byreal` / `mantle-sepolia`. |
| `BYREAL_MODE` | byreal adapter | `mock` (default) or `real`. |
| `AGENT_URI_MODE` | register-agents | `data` (inline) or `hosted` (AGENT_URI_BASE). |
| `AGENT_URI_BASE` | register-agents | Required when `AGENT_URI_MODE=hosted`. |
| `DEMO_SEED` | demo-runner | Stable seed for scenario picker + proof hashes. |
| `FORCE_SCENARIO` | demo-runner | Force a specific scenario id (e.g. `validator-rejects`). |
| `POLL_INTERVAL_MS` | poll-chain | Refresh interval for chain snapshots (default 30000). |
| `POLL_ONCE` | poll-chain | `1` = snapshot once and exit. |
| `WATCH_LOOKBACK_BLOCKS` | watch-events | Number of past blocks to backfill on startup (default 5000). |
| `WATCH_INTERVAL_MS` | watch-events | Subscribe poll interval (default 12000). |
| `WATCH_MAX_EVENTS` | watch-events | Cap on stored events (default 100). |
| `WATCH_ONCE` | watch-events | `1` = backfill once and exit. |
| `WATCH_FIXTURE` | watch-events | `1` = write a synthetic events.json without RPC (offline preview). |
| `MAX_ACTION_VALUE` | deploy-treasury | Constructor arg in wei (default 1 ether). |
| `DRY_RUN` | scripts | `1` = print only. |

## Recommended ramp

1. **Local demo only** (default state):
   ```bash
   npm install
   npm run demo
   npm run dev
   ```
2. **Real ERC-8004 registration**:
   - Generate a mnemonic, fund the first five derived addresses with Mantle
     Sepolia MNT (faucet).
   - `cp .env.example .env`, paste mnemonic + RPC URL.
   - `DRY_RUN=1 npm run register` (sanity check).
   - `npm run register` (real broadcast).
   - `npm run verify` (confirm chain state).
3. **Real on-chain execution**:
   - Deploy `AgenticTreasury` (see `contracts/README.md`), set
     `TREASURY_ADDRESS`.
   - `EXECUTOR=mantle-sepolia npm run demo`.
   - The demo runner emits a real `executeApprovedAction` tx whose hash
     appears in the dashboard.
4. **Real reputation feedback**:
   - `npm run feedback`.
   - `npm run verify` again to see `feedbackCount` increase per agent.

## Foundry contracts

```bash
cd contracts
forge install foundry-rs/forge-std --no-commit
forge build
forge test -vv
```

Tests cover: happy path, replay protection (nonce + actionId), expired
deadline, value cap, data-hash mismatch, signature forgery, and risk-officer
rotation invalidating prior signatures.

## Troubleshooting

- **Vite picks port 5175 not 5173**: ports 5173/5174 are commonly held by
  other local Vite projects on this machine. The dev server config pins 5175.
- **`viem` chain name**: viem exports `mantleSepoliaTestnet` (chain id 5003),
  re-exported as `mantleSepolia` inside the executor + scripts.
- **Demo writes `data:` URI registrations**: that means `AGENT_URI_MODE=data`
  is the default. Switch to `AGENT_URI_MODE=hosted` once you deploy the
  dashboard publicly.
