# ERC-8004：Trustless Agents 标准

## 一句话

ERC-8004 是 **agent 版的 ENS + 大众点评**：用三个链上 registry 让 agent 跨组织被发现、被打分、被验证，不需要预先信任。

## 关键事实

- **作者**：Marco De Rossi (MetaMask) / Davide Crapis (Ethereum Foundation) / Jordan Ellis (Google) / Erik Reppel (Coinbase)
- **创建**：2025-08-13
- **主网上线**：2026-01-29
- **依赖**：ERC-155 / ERC-712 / **ERC-721** / ERC-1271
- **状态**：Draft（仍在 standards track）

## 三个 Registry

### 1. Identity Registry（ERC-721 + URIStorage）

每个 agent 是一枚 NFT：
- `tokenId` → `agentId`
- `tokenURI` → `agentURI`，指向一个 JSON 注册文件（IPFS / HTTPS / data URI 都行）
- 完整 agent 标识：`eip155:<chainId>:<registry>:<agentId>`

JSON 注册文件标准结构（核心字段）：
```json
{
  "type": "https://eips.ethereum.org/EIPS/eip-8004#registration-v1",
  "name": "myAgentName",
  "description": "what it does, pricing, interaction methods",
  "image": "https://.../agentimage.png",
  "services": [
    {"name": "A2A", "endpoint": "https://agent.example/.well-known/agent-card.json", "version": "0.3.0"},
    {"name": "MCP", "endpoint": "https://mcp.agent.eth/", "version": "2025-06-18"},
    {"name": "ENS", "endpoint": "vitalik.eth"},
    {"name": "DID", "endpoint": "did:method:foobar"}
  ],
  "x402Support": false,
  "active": true,
  "registrations": [
    {"agentId": 22, "agentRegistry": "eip155:5000:0x8004A169..."}
  ],
  "supportedTrust": ["reputation", "crypto-economic", "tee-attestation"]
}
```

### 2. Reputation Registry

标准化的 feedback signals 接口。链上做基础聚合，链下可叠加 ML 评分、insurance、auditor 网络。

### 3. Validation Registry

通用 hooks：让 staker re-execute、zkML verifier、TEE oracle、可信法官等独立 validator 写回结果。

## 核心合约接口

```solidity
// 注册 agent（最简）
function register() external returns (uint256 agentId);
function register(string agentURI) external returns (uint256 agentId);
function register(string agentURI, MetadataEntry[] calldata metadata) external returns (uint256 agentId);

// 更新 URI
function setAgentURI(uint256 agentId, string calldata newURI) external;

// 自定义元数据
function getMetadata(uint256 agentId, string memory metadataKey) external view returns (bytes memory);
function setMetadata(uint256 agentId, string memory metadataKey, bytes memory metadataValue) external;

// agent 收款钱包
function setAgentWallet(uint256 agentId, address newWallet, uint256 deadline, bytes calldata signature) external;
function getAgentWallet(uint256 agentId) external view returns (address);
function unsetAgentWallet(uint256 agentId) external;
```

事件：
- `Transfer`（来自 ERC-721）
- `Registered(agentId, agentURI, owner)`
- `URIUpdated(agentId, newURI, updatedBy)`
- `MetadataSet(agentId, indexedKey, key, value)`

## 域名验证（可选但加分）

agent 可以把 `https://{endpoint-domain}/.well-known/agent-registration.json` 部署成 server-known 文件，包含：
```json
{ "registrations": [{"agentId": 22, "agentRegistry": "eip155:5000:0x8004..."}] }
```
谁拉到这个文件就视为 endpoint 域名验证通过。

## 对你的意义（黑客松专用）

1. **黑客松要求所有 agent 通过 ERC-8004 拿身份 NFT**——你必须实现 register 逻辑。
2. 因为合约已经部署在 Mantle，**你不用花一分钱去部 registry，只需调用 register()**——成本极低。
3. **真正能拉开差距的是 reputation/validation 部分**——绝大多数选手只会做 Identity，做出 reputation 反馈闭环或 validator 接入是大加分项。
4. **多 agent 协作 + 各自身份** 是天然的展示场景，pitch 时 demo 几个不同 agent 在链上互相调用、留下声誉记录，视觉冲击力极强。
