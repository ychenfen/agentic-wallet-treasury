# 赛道 4：Consumer & Viral DApps

## 官方定义

> Gamified trading interfaces and shareable consumer applications.

## 关键词解构

- **gamified trading interfaces**：交易当游戏玩
- **shareable consumer apps**：能在 Twitter / 朋友圈裂变的玩具型 dApp

## 评审会重点看

1. 病毒系数（K-factor）——产品自己能不能拉新
2. UI / 视觉冲击力
3. 是否真的有 agent 介入而不是纯前端噱头
4. 留存和参与曲线（如果你能在 demo 前先跑出几百 DAU 会非常加分）

## 项目方向

| 方向 | 一句话 | 难度 |
|---|---|---|
| **AI Trading Battle Royale** | 用户出题（"用 1000U 24 小时跑赢 ETH"），多个 AI agent 接单，胜者上 leaderboard，败者出局 | 中高 |
| **Agent Pet** | 每个用户在 ERC-8004 上养一只 agent，喂资金，它会自己打理收益，胖瘦是收益情况的可视化 | 中 |
| **Predict-and-trade meme** | 用户分享一个预测帖（"BTC 周五 6 万"），AI agent 自动按预测开仓，结果上链分享 | 中 |
| **Onchain Tamagotchi DEX** | 把 LP / yield 包装成养成游戏，agent 自动操作，用户只看角色升级 | 中 |

## 竞争烈度

🟡 **中**。这条赛道两极分化——做 demo 的多，但能做出真实留存的少。

## 你用 Codex/Claude Code 能打到的程度

- **天花板**：赛道前 5（视觉好看 + 一个 viral hook）
- **杀手锏**：找一个真正"有人愿意分享"的钩子。这部分 AI 帮不了你，是产品直觉问题
- **建议**：如果你产品感强、有美术资源 → 选；如果没有 → 不选

## 必读资源

- Animoca Brands 投资组合（看它喜欢什么类型）
- Farcaster Frames 案例（病毒式 dApp 的当代教科书）
